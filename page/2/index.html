Mysql数据库

2022/7/11

数据库是按照数据的结构，存储和管理数据的仓库

数据库分为 关系型数据库 和 非关系型 数据库

关系性数据库：sql数据库容易理解 使用方便 易于维护，缺点海量数据读写低，难以扩展

非关系性数据库：nosql ，not only sql就是为了加快客户端访问速度时出现的

mysql存储原理

整体架构：

1.连接池：最上层负责和客户端进行连接

2.sql接口：当sql语句进入mysql，会先到sql接口中，这一层是封装层

3.解析器：这一层负责将sql语句进行拆分验证

4.优化器：对sql查询的结果优化处理，产生多种执行计划。最终数据库会选择最优话的方案

5.缓存：对要查询的sql语句进行hash后缓存

6.存储引擎：mysql有很多种存储引擎，每一种存储引擎有不同的特性，他们负责组织文件存放位置，访问方法

下载安装mysql

点击DOWNLOADS，选择mysql community社区版根据需求下载

mysql官网有三种安装方式：

1.压缩包安装

2.压缩包自带boost安装

3.rpm包安装

安装好以后尝试进入数据库，如果可以进入说明安装成功

6.索引

索引的好处：1、帮助用户提高查询速度. 2、利用索引的唯一性来控制记录的唯一性. 3、可以加速表与表之间的连接. 4、降低查询中分组和排序的时间

索引的坏处：1、存储索引占用磁盘空间 2、执行修改操作（insert、delete、update）产生索引维护

索引是用什么数据结构的？

B+树

b+树，是b树的一种变体，查询性能更好

mysql有哪些存储引擎，默认innoDB与 MylSAM区别

InnoDB、MyISAM、Memory、Merge、Archive、CSV、BLACKHOLE 等

  1. InnoDB支持事务，MyISAM不支持，对于InnoDB每一条SQL语言都默认封装成事务，自动提交，这样会影响速度，所以最好把多条SQL语言放在begin和commit之间，组成一个事务；
  2. InnoDB支持外键，而MyISAM不支持。对一个包含外键的InnoDB表转为MYISAM会失败；

索引有哪些类型，一定要把组合索引说出来，然后会问组合索引规则即最左原则

​ 普通索引index :加速查找

唯一索引

主键索引：primary key ：加速查找+约束（不为空且唯一）

唯一索引：unique：加速查找+约束 （唯一）

联合索引

-primary key(id,name):联合主键索引

-unique(id,name):联合唯一索引

-index(id,name):联合普通索引

全文索引fulltext :用于搜索很长一篇文章的时候，效果最好。

空间索引spatial :了解就好，几乎不用

7.SQL优化或者说什么时候索引失效

避免使用*通配符“%”或者“_”作为查询字符串的第一个字符
在设计表时，把索引列设置为NOT NULL
Where子句中避免在索引列上使用计算
索引列上进行函数操作，索引失效
用>= 代替 > 尽量不使用 <> 或者!=
使用EXISTS代替 in 用not EXISTS 代替 not in

1、like的模糊查询以%开头
2、条件中用or，即使其中有条件带索引，也不会使用索引查询
3、查询条件如果是字符串，不加引号会使索引失效
4、not null 不会使用索引， is null 会使用索引
5、使用 <> 或者!=
6、in not in
7、在索引列上使用计算

DNS解析过程

2022/7/11

网站的访问过程 DNS的解析过程

1.例如访问www.baidu.com 先查看本地电脑的DNS缓存

2.如果本地没有对应的DNS缓存，客户端会查看本地的host文件

3.如果没有，客户端会将域名交给本地填写的DNS，以上称为 递归查询

4.如果本地DNS，没有找到对应的域名或IP就会将请求交给根DNS

5.根DNS服务器知道 .com服务器

.com服务器 认识 baidu.com服务器，称为迭代查询

如果网站有备案拿肯定会有ip地址，如果没有就是没备案

如果找到目标则交给本地DNS，本地DNS返回给客户端

DNS的端口号是： 53

协议是： TCP UDP 相互传输时使用TCP协议，其他的时候使用 UDP

Gitlab与jenkins构建持续集成（CI）环境

2022/7/11

Gitlab 与Github

是一个代码托管平台

起初开发人员都会将代码放到类似

git svn 版本控制器

使用git存放代码，如需查看代码还需要将代码拉取到本地

随着时间的发展，开发人员逐渐使用gitlab 和 github

不管是gitlab还是github他们的底层都是git开发的，两者都能存放代码

gitlab和github都能通过浏览器去访问

gitlab与github的区别

如果代码存放到github上那么别人是可以看到你的项目，github也有私有的仓库，但是一个账户只能创建4个，超出则需要额外收费

存放到gitlab别人则看不到，因为gitlab是存放到你本地服务器的

gitlab有两个版本

gitlab-ce社区版(免费) gitlab-ee企业版(收费) 基本上在公司也是用免费的社区版

jenkins

开发吧代码放到gitlab上，jenkins去gitlab上拖到本地，然后把代码发送到目标浏览器

标题CI的意思是 devops

​ dev=开发 ops=运维 连起来就是运维开发

持续集成简称CI :持续集成是指开发在代码的开发过程中可以频繁的将代码部署集成到gitlab主干，并进行自动化测试

持续交付 : 在持续集成的基础上，将代码部署到预生产环境 称为CD

持续部署：在持续交付的基础上，把代码部署到生产环境的过程自动化称为CD
